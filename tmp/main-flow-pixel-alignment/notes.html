06 받은 고민

  • loaded/success branch만 화면 전용 393px x 852px 캔버스로 교체했다. loading/error/empty branch는 유지했다.
  • 카드는 전체 클릭 가능한 실제 button을 유지하고, 건너뛰기 버튼은 event.stopPropagation(), onPass, disabled/processing 상태를 유지한다.
  • 상단 프로필/알림은 preview 및 chrome-neutral 경로를 위한 시각 전용 요소다. 기존 production handler가 없으므로 새 navigation behavior를 만들지 않았다.
  • preview harness fixture 문자열은 참조 Component.tsx의 한국어 문구를 production prop shape에 맞게 재사용했다. production route에는 fixture 문자열이 없다.
  • production에서는 카드 개수를 맞추기 위한 fabricated card를 만들지 않는다. 실제 props.items가 1개면 1개 카드만 보이며, 이는 reference PNG와의 남은 시각 차이로 기록한다.

07 질문 작성

  • WriteWorryContainer.tsx가 route back/header chrome 소유자라서 해당 파일을 393px x 852px 캔버스 소유자로 바꿨다.
  • WriteFormScreen.tsx의 kind === "write-worry" branch만 전용 절대 좌표 레이아웃으로 바꿨다. write-reply branch는 유지했다.
  • textarea는 실제 textarea이며 value, onDraftChange, maxLength, disabled/processing 상태를 그대로 사용한다.
  • AI 안심 필터 정보는 product behavior를 삭제하지 않고 DOM에 남겼으나, 참조 캡처 영역과 충돌하지 않도록 캔버스 아래쪽에 배치했다. validation/moderation 메시지는 production 상태가 요구할 때만 표시된다.
  • preview harness fixture는 참조 Component.tsx의 placeholder와 CTA 문구를 사용했다. production route는 container의 real draft state를 사용한다.

08 답변 확인

  • loaded 상태의 variant === "received-answer-detail"에만 ReceivedAnswerDetailFrame을 적용했다. my-answer-detail, MyAnswerFeedbackPanel, generic comment behavior는 기존 분기를 유지한다.
  • feedback 버튼은 기존처럼 클릭 시 onFeedbackChange 다음 onFeedbackSubmit을 호출한다.
  • comment UI는 기존 production 상태가 요구하는 submitted-like-without-comment 조건에서만 렌더링한다.
  • preview harness의 원문 고민 한국어 문구는 참조 Component.tsx에서 가져왔고, 답변 본문은 현재 참조가 lorem ipsum을 사용하므로 같은 줄 수의 대체 문자열을 fixture에 사용했다. production route는 real detail props를 사용한다.

Production shell integration fix

  • 첫 구현은 393x852 정렬 캔버스를 기존 MobileAppShell route body 안에 중첩했다. 그 결과 global fixed Qling header, main의 pt-24, shell horizontal padding/max-width, App-level 06 route title row, real fixed BottomNavigation이 그대로 남았다.
  • MobileAppShell에 presentation-only frameMode?: "default" | "pixel-aligned"를 추가했다. 기본값은 default라서 비대상 route 동작은 그대로 유지된다.
  • frameMode="pixel-aligned"에서는 main이 mx-auto min-h-[852px] w-[393px] p-0만 사용한다. max-w-2xl, shell-x padding, pt-24, bottom scroll padding을 적용하지 않는다. CSS transform scaling은 사용하지 않는다.
  • App.tsx는 06(답변하기, received_worries), 07(write_worry), 08(answer_check, read_received_reply, received_answer_detail)에서만 usesPixelAlignedFrame을 켠다.
  • usesPixelAlignedFrame이 true일 때 App shell은 global authenticated header를 넘기지 않고, real BottomNavigation도 넘기지 않으며, hasBottomNavigation={false}와 frame-neutral main을 사용한다.
  • 06 route branch는 pixel-aligned mode에서 App-level 답변하기 title row와 interest badge를 렌더링하지 않는다. 06 내부 시각 bottom chrome만 남기고 real fixed bottom navigation은 mount하지 않아 duplicate navigation chrome이 없다.
  • 07 write-worry canvas는 global header/bottom nav 아래에 떠 있는 child가 아니라 frame-neutral main의 직접 body로 렌더링된다. 내부 back button, textarea, submit behavior는 그대로 유지했다.
  • 08 received-answer-detail canvas도 frame-neutral main의 직접 body로 렌더링된다. my-answer-detail branch는 기존 shell/panel behavior를 유지한다.
  • 수정은 route-frame presentation logic에 한정했다. route selection, auth guard, data loading, handlers, contracts, mapping modules, services, Firestore, server code는 변경하지 않았다.

Production functionality reintegration

  • production routes는 real container props를 사용한다. fixture strings and objects는 tmp/main-flow-pixel-alignment/preview.tsx에만 존재한다.
  • ReceivedWorriesScreen은 loaded branch에서 props.items.map((item, index) => ...)만 사용한다. 각 card는 item.deliveryId, item.worryId, item.category, item.receivedAt.isoValue, item.receivedAt.label, item.bodyText ?? item.previewText, item.isUnread, props.passingDeliveryIds로 렌더링/동작한다.
  • 06의 decorative internal bottom chrome은 production screen에서 제거했다. 대신 BottomNavigation presentationMode="pixel-aligned"를 App shell이 같은 393px frame 안에 렌더링하며, 기존 onSelectTab과 onCentralAction handlers를 그대로 사용한다.
  • 06 상단 profile/bell chrome은 기존 production handler가 없는 visual-only chrome이다. 새 profile 또는 notification navigation behavior를 만들지 않았고, product navigation은 기능형 bottom navigation으로 유지한다.
  • 07은 real draft.value, onDraftChange, submitDisabledReason, isProcessing, onPublish, validation/moderation 상태를 사용한다. AI 필터 안내는 DOM에 유지하되 reference capture 영역 밖에 배치했다.
  • 08은 real props.originalWorry, props.reply, props.existingFeedback, props.selectedFeedback, comment validation/moderation/draft props를 사용한다. feedback buttons는 기존처럼 onFeedbackChange 다음 onFeedbackSubmit을 호출한다.
  • focused tests를 추가해 06/07/08이 supplied props text를 렌더링하고, production screen HTML에 preview-only strings such as one two three, reference card text, or Lorem ipsum이 나타나지 않도록 확인했다.

Shared bottom navigation alignment

  • 이전 수정은 기능형 BottomNavigation을 393px frame 안에 넣었지만, old production design인 fixed/grid/lucide/central text pill/border shadow style을 그대로 유지했다.
  • BottomNavigation에 presentation-only variant?: "default" | "pixel-aligned"를 추가했다. default variant는 기존 fixed viewport nav, 3-column grid, lucide icons, central 고민 작성 pill을 그대로 유지한다.
  • pixel-aligned variant는 shared UI 한 곳에서만 reference bottom bar geometry를 구현한다. production screens에는 screen-local fake nav 또는 aria-hidden replacement nav를 렌더링하지 않는다.
  • pixel variant는 absolute left:0 bottom:0 width:393px height:112px frame 안에서 base cream band, side tab buttons, center owner-tab hit area, and orange central write action을 배치한다.
  • 측정 기준은 06 bottom nav bbox (0,740,393,852), active item (135,755,260,835), 08 bottom nav/chrome (0,760,393,852)를 시작점으로 삼았다. 구현 CSS는 selected bg left:135 top:15 width:125 height:80, center action left:149 top:24 width:95 height:59이다.
  • 각 nav item은 functional button이다. side tabs call onSelectTab(tab), center owner-tab background calls onSelectTab(ownerTab.tab), orange center action calls onCentralAction. Active tab state는 tab buttons에 aria-current="page"로 반영된다.
  • 중앙 orange shape는 reference의 large center item을 write-worry central action으로 해석했다. 그 주변 cream selected background는 central action의 owner tab인 나의 고민 tab hit area로 유지해 tab navigation도 잃지 않았다.
  • 남은 mismatch: reference SVG icon path와 production CSS-drawn icon glyph가 완전히 동일하지는 않다. 그러나 old lucide icon/text pill/grid visual은 pixel variant에서 제거했고, navigation behavior는 보존했다.
